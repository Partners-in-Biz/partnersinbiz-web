# Evidence Ledger

Agents must leave proof behind. This skill defines the minimum standard for
evidence attached to Partners in Biz work.

## Evidence Types

- source document/spec/version/approval ids
- research item/source ids
- pull request and commit URLs
- preview or production deployment URLs
- campaign, social post, ad, SEO, invoice, CRM, or report record ids
- test output summaries
- smoke-test results
- approval comments
- client-visible deliverable links

## Rules

- Prefer platform record ids and canonical URLs over vague prose.
- Record enough detail for another agent or Peet to verify the result without chat history.
- Do not store secrets or private tokens in task comments or evidence artifacts.
- Mark missing evidence as a blocker when the task cannot be verified.
