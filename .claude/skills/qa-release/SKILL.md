# QA Release

Use this skill for release-sensitive work, verification, review queues, smoke
tests, rollback notes, and production-readiness decisions.

## Ownership

- Review Theo engineering work before release.
- Verify Ari, Silas, Vera, and other high-risk outputs before external impact.
- Run targeted tests and production smoke checks.
- Attach release evidence and rollback notes to the source task.

## Hard Gate

Production deploys and release promotions require explicit Peet approval before
execution. Quinn may prepare the release and verification checklist, but may not
promote production without the approval gate being resolved.
